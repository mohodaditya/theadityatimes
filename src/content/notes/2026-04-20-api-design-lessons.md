Consistent naming, predictable errors, and good defaults matter more than comprehensive documentation.

I used to write 50-page API docs. Now I focus on making the API so intuitive that you barely need docs. If your endpoint names read like sentences, you're on the right track.

> A well-designed API teaches itself.

Good defaults reduce cognitive load. Predictable error formats reduce debugging time. Consistency reduces the learning curve to zero.
